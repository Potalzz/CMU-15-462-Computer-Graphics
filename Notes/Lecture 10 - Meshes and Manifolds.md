## Meshes and Manifolds

### Meshes
- 표면은 물체의 내부가 아니라 경계(껍질)에 해당하는 2차원 형상
- 컴퓨터 그래픽스에서 다루는 많은 대상은 표면 모델로 충분하다는 전제를 둠.

### Manifolds
- 매니폴드 표면은 임의의 점을 충분히 확대했을 때 국소적으로 2차원 좌표격자를 놓을 수 있는 성질을 가짐.  
- 즉, 국소적으로는 평면과 같은 2차원 구조를 가져야 함.

### 비매니폴드 예시
- 모든 형상이 매니폴드인 것은 아님.  
- 특정 점(예: 두 표면이 한 점에서 접합되는 형태)에서는 아무리 확대해도 정상적인 2차원 격자 구조가 성립하지 않음.
- 용접된 접합부처럼 한 점에서 표면이 3개 이상 방향으로 갈라지면 비매니폴드가 됨.

## 폴리곤 메시에서의 매니폴드 조건

### 팬은 있고 핀은 없음
- 매니폴드 폴리곤 메시의 핵심 암기법은 팬은 있고 핀은 없음임.  
- 조건 1. 모든 내부 에지는 정확히 두 개의 폴리곤에 포함되어야 함.  
  - 하나의 에지에 3개 이상 폴리곤이 붙으면 비매니폴드 에지(핀)가 됨.  
- 조건 2. 한 정점에 인접한 폴리곤들은 하나의 연속된 고리(팬)로 연결되어야 함.  
  - 한 정점 주변 폴리곤이 둘 이상의 분리된 묶음으로 나뉘면 비매니폴드 정점이 됨.
  - ![](../assets/images/Pasted%20image%2020260303120906.png)

### 경계가 있는 매니폴드
- 경계는 표면이 끝나는 부분임.  
- 매니폴드 경계의 성질은 다음과 같음.  
  - 국소적으로는 반원판 형태로 보임.  
  - 전역적으로는 각 경계가 닫힌 루프를 이룸.  
- 폴리곤 메시 관점의 조건은 다음과 같음.  
  - 경계 에지는 정확히 한 개의 폴리곤에만 포함됨.  
  - 경계 정점 주변 폴리곤은 고리가 아니라 띠(스트립) 형태로 배열됨.

## 매니폴드 가정을 하는 이유
- 목표는 자료구조와 알고리즘을 단순하고 효율적으로 만드는 것임.  
- 이미지에서 픽셀 격자를 사용해 단순성과 효율을 얻는 것과 같은 동기임.![](../assets/images/Pasted%20image%2020260303120940.png)
- 일반적인 그래픽스/애니메이션/시뮬레이션 대상은 매니폴드 가정으로도 충분히 표현 가능한 경우가 많음.

## 메시 자료구조 설계 관점: 배열 vs 연결 리스트(Linked List)
- 리스트 저장 문제를 배열과 연결 리스트로 대비함.  
  - 배열은 임의 접근이 빠르고 메모리 접근이 연속적이어서 효율적임.  
  - 연결 리스트는 삽입/삭제가 쉬우나 접근과 캐시 효율이 불리함.  
- 이후 메시 자료구조들도 이 트레이드오프를 반복적으로 가짐.

## 폴리곤 메시 표현 방식들

### Polygon Soup
- 각 삼각형에 대해 3개 정점 좌표를 그대로 저장하는 방식임.  
- 연결성 정보(인접 관계)를 별도로 저장하지 않음.  
- 장점은 구현이 매우 단순하다는 점임.  
- 단점은 정점 데이터 중복이 크고, 이웃 탐색 등 고급 연산이 거의 불가능하다는 점임. [[슬라이드 18: Polygon Soup 장단점]]  

### Adjacency List
- 정점 좌표 리스트와 폴리곤 인덱스 리스트를 분리해서 저장함.  
  - 기하(geometry)와 연결성(connectivity)을 분리하는 첫 단계임.  
- 특정 정점에 닿는 폴리곤을 찾는 질의는 폴리곤 전체를 선형 탐색해야 함.  
- 폴리곤 수가 매우 큰 경우(예: 수억~수십억)도 전체 탐색 비용이 되어 비효율적.

### 발생(incidence) 행렬
- 누가 이웃인지 알고 싶으면 이웃 정보를 행렬로 저장하자는 관점임.  
- 대표적으로 다음 행렬을 사용함.  
  - 정점-에지 발생 행렬(Vertex–Edge incidence).  
  - 에지-면 발생 행렬(Edge–Face incidence).  
- 행렬 원소는 접촉하면 1, 아니면 0으로 정의됨.  
- 대부분의 원소는 0이므로 희소 행렬로 저장해야 실용적임.  
- 장점은 특정 원소의 이웃 접근이 행/열 조회로 빠르다는 점임.  
- 단점은 인덱스 기반의 고정 구조라 연결성 변경이 불편하다는 점임.  
- 추가 장점은 비매니폴드 연결도 표현 가능하다는 점임.

### Sparse Matrix 자료구조
- Sparse Matrix를 저장하는 대표 구조는 다음과 같음.  
  - (행, 열) → 값의 연관 배열 방식임. 조회/갱신이 쉬우나 연산(곱셈 등)이 불리함.  
  - 행별 연결 리스트 배열 방식임. 개념은 단순하나 접근과 메모리 지역성이 불리함.  
  - 압축 열 저장(Compressed Column, CSC) 방식임. 값 배열, 행 인덱스 배열, 열 포인터 배열로 구성됨.  
    - 수정은 불편하나 희소 선형대수 연산에 매우 유리함.  
- 실무적으로는 구축은 쉬운 구조로 하고, 계산 전 CSC로 변환하는 흐름이 흔함.

## Halfedge 자료구조 (Linked-list-like)

### 기본 아이디어
- 메시 요소 간의 인접 관계를 최소 포인터로 저장하고, 포인터 추적으로 필요한 정보를 복원하는 방식.
- 모든 에지를 방향을 가진 두 개의 halfedge로 분해한다.
- 각 halfedge가 다음 정보를 가진다.
  - `twin` : 같은 에지의 반대 방향 halfedge
  - `next` : 같은 면에서 다음 halfedge
  - `vertex` : halfedge의 시작 정점(관례상 from vertex)
  - `edge` : 소속 에지
  - `face` : 소속 면
- 정점/에지/면은 각각 자신과 인접한 halfedge 하나만 참조함.![](../assets/images/Pasted%20image%2020260303121620.png)

### Halfedge mesh 순회(traversal)
- 면의 모든 정점을 방문하는 방법은 `next`를 따라 한 바퀴 도는 방식.
- 정점 주변의 이웃을 방문하는 방법은 `twin -> next`를 반복하는 방식.

예시 코드.
```cpp
// face의 정점들을 순회함.
Halfedge* h = f->halfedge;
do {
    // h->vertex 사용 가능함.
    h = h->next;
} while (h != f->halfedge);

// vertex 주변을 순회함.
Halfedge* h2 = v->halfedge;
do {
    h2 = h2->twin->next;
} while (h2 != v->halfedge);
```

### Halfedge 연결성의 불변조건
- 단순화한 halfedge( `next`, `twin` 만 보유 )에서도 다음 조건이 요구됨.  
  - `twin->twin == this` 임.  
  - `twin != this` 임.  
  - 모든 halfedge는 누군가의 `next`로부터 도달 가능해야 함.  
- 이 조건들을 만족하면 `next` 루프는 면을, `twin` 쌍은 에지를, `next->twin` 루프는 정점을 구성하게 됨.  
- 따라서 halfedge 구조는 기본적으로 매니폴드 연결만 표현하게 됨.

## 연결성(connectivity)과 기하(geometry)의 분리

### 핵심 구분
- 매니폴드 조건(팬/핀 조건)은 정점 좌표가 아니라 연결성에 대한 조건임.  
- 즉, 연결성이 매니폴드라도 정점 좌표 배치가 나쁘면 자기교차 등 기하적으로 문제가 생길 수 있음.

### 디버깅 관점
- 화면에서 메시가 이상해 보인다고 해서 항상 연결성 오류인 것은 아님.  
- 매우 얇은 삼각형 등 기하 품질 문제일 수도 있음.

## Halfedge 기반 로컬 편집 연산

### Halfedge가 편집(edit)에 강한 이유
- 연결 리스트가 삽입/삭제에 유리한 것과 같은 이유로, halfedge는 로컬 연결성 수정에 유리함.  
- 요소를 할당/해제하고, 포인터를 재배치하여 로컬 연산을 구현함.  
- 구현 시 매니폴드 불변조건을 유지해야 함.
- ![](../assets/images/Pasted%20image%2020260303122127.png)

### 에지 플립(edge flip)
- 두 삼각형이 공유하는 에지를 반대 대각선으로 교체하는 연산임.  
- 포인터 재배치가 핵심이며, 요소 생성/삭제는 없음.  
- 두 번 연속 적용하면 원래 연결로 되돌아감.  
- 구현에 따라 동일 연결을 표현하더라도 포인터 배치가 달라질 수 있음.

### 에지 스플릿(edge split)
- 에지 중점에 새 정점을 추가하고 주변을 재삼각분할하는 연산임.  
- 새 정점, 새 에지, 새 면, 새 halfedge 등 요소 생성이 필요함.  
- 포인터 재배치량이 큼.

### 에지 콜랩스(edge collapse)
- 에지의 두 정점을 하나로 합치고 인접 요소를 제거하는 연산임.  
- 요소 삭제와 포인터 재배치가 모두 필요함.  
- 인접 리스트 기반 구현은 영향을 받는 요소 탐색과 리스트 편집이 번거로움.

## Halfedge 대안과 자료구조 선택

### 대안 자료구조
- winged edge, corner table, quadedge 등 다양한 변형이 존재함.  
- 공통점은 이웃 정보를 저장하고 포인터/인덱스 추적으로 순회하는 방식.
- 단점은 추가 저장공간과 메모리 접근 효율이 떨어짐.
- 장점은 어떤 구간/근처만 보는 작업이나 순회가 빠르고 로컬 편집이 직관적이다.

### 비교 요약
![](../assets/images/Pasted%20image%2020260303122758.png)
- 인접 리스트는 이웃 접근이 느리며 편집도 불리한 편임.  
- 발생 행렬은 이웃 접근이 빠르나 연결성 변경이 불리함. 비매니폴드도 표현 가능함.  
- halfedge는 이웃 접근과 편집에 유리하나 비매니폴드 연결은 기본적으로 표현 불가함.  
- 결론은 문제에 맞는 자료구조 선택이 핵심

---

### 매니폴드 메시 판정 체크
- 내부 에지는 정확히 두 개의 면에 포함되는지 확인
- 정점 주변 면들이 하나의 팬(단일 루프 또는 경계에서는 단일 스트립)을 이루는지 확인
- 경계 에지는 정확히 한 개의 면에 포함되는지 확인

### halfedge 구현 불변조건 체크
- `he->twin->twin == he` 인지 확인함.  
- `he->twin != he` 인지 확인함.  
- 모든 halfedge가 어떤 halfedge의 `next`로부터 도달 가능한지 확인.

## 용어 정리
- 표면(surface) - 물체의 경계로 해석되는 2차원 형상
- 연결성(connectivity) - 정점/에지/면이 어떻게 인접하는지에 대한 정보
- 기하(geometry) - 정점 좌표 등 실제 위치 정보
- 매니폴드(manifold) - 국소적으로 2차원 좌표계를 둘 수 있는 표면/연결성
- halfedge - 방향을 가진 에지 절반 단위이며, 메시 순회와 편집의 핵심 요소
